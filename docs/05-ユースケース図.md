# ユースケース図

## 全体図

```mermaid
flowchart TB
    subgraph 一般ユーザー
        U((ユーザー))
    end

    subgraph 管理者
        A((管理者))
    end

    subgraph システム
        UC1[質問して回答を受ける]
        UC2[ロールを設定する]
        UC3[ナレッジを登録する]
        UC4[ナレッジを削除する]
        UC5[ベクトルを再作成する]
    end

    U --> UC1
    A --> UC2
    A --> UC3
    A --> UC4
    A --> UC5
```

## ユースケース一覧

| ID | ユースケース | アクター | 説明 |
|----|--------------|----------|------|
| UC1 | 質問して回答を受ける | ユーザー | LINE でテキストを送信し、RAG/LLM による回答を受け取る |
| UC2 | ロールを設定する | 管理者 | 管理画面で Bot のキャラクター・振る舞い（System Instruction）を設定する |
| UC3 | ナレッジを登録する | 管理者 | 想定Q&A、カテゴリ、ソースを登録し、埋め込みを保存する |
| UC4 | ナレッジを削除する | 管理者 | 登録済みナレッジを選択して削除する |
| UC5 | ベクトルを再作成する | 管理者 | 全件を再埋め込みしてベクトルDBを更新する |

## UC1: 質問して回答を受ける（詳細）

```mermaid
flowchart TD
    Start([ユーザーが LINE でメッセージ送信]) --> A{署名検証}
    A -->|失敗| End1([401 エラー])
    A -->|成功| B{メッセージタイプ}
    B -->|text 以外| End2([無視])
    B -->|text| C{重複?}
    C -->|重複| End2
    C -->|新規| D{ナレッジ件数}
    D -->|0件| E[Gemini に直接質問]
    D -->|1件以上| F[埋め込み→類似度検索]
    F --> G{検索結果}
    G -->|あり| H[プロンプト構築→Gemini 生成]
    G -->|なし & フォールバック| E
    G -->|なし & フォールバックOFF| I[分かりかねます]
    E --> J[LINE に返信]
    H --> J
    I --> J
    J --> End3([完了])
```

## UC3: ナレッジを登録する（詳細）

```mermaid
flowchart TD
    Start([管理者が登録画面を開く]) --> A[想定質問・回答を入力]
    A --> B[カテゴリ・ソースを入力（任意）]
    B --> C[「登録」クリック]
    C --> D{必須項目チェック}
    D -->|NG| E[エラー表示]
    D -->|OK| F[Gemini で埋め込み生成]
    F --> G[KnowledgeStore.upsert]
    G --> H[成功表示]
```

## アクターとシステム境界

```mermaid
flowchart LR
    subgraph 外部
        User[ユーザー]
        Admin[管理者]
    end

    subgraph LINE_RAG_Bot["LINE RAG Bot システム"]
        subgraph API["FastAPI"]
            Webhook[/webhook]
        end
        subgraph UI["Streamlit"]
            RoleTab[ロールタブ]
            KnowledgeTab[ナレッジタブ]
        end
    end

    User -->|メッセージ送受信| Webhook
    Admin -->|ロール編集| RoleTab
    Admin -->|Q&A登録/削除/再埋め込み| KnowledgeTab
```
