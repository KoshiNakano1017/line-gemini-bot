# LINE RAG Bot アプリケーション仕様

## 概要

LINE で受けた質問に対して、管理画面で設定した **ロール（System Instruction）** と、登録済みの **ナレッジ（RAG）** を元に Gemini が回答するチャットボットです。

## 機能一覧

| 機能 | 説明 | 対象 |
|------|------|------|
| 質問応答 | ユーザーが LINE で送ったテキストに回答 | 一般ユーザー |
| ロール設定 | ボットのキャラクター・振る舞いを定義 | 管理者 |
| ナレッジ登録 | 想定Q&A を登録し、RAG 検索の対象にする | 管理者 |
| ナレッジ削除 | 登録済みのナレッジを削除 | 管理者 |
| ベクトル再作成 | 全件を再埋め込みしてベクトルを更新 | 管理者 |

## 技術仕様

### API エンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/health` | ヘルスチェック |
| GET | `/debug/config` | 設定情報のデバッグ表示 |
| GET | `/debug/models` | 利用可能な Gemini モデル一覧 |
| POST | `/webhook` | LINE Webhook（メッセージ受信） |

### RAG 仕様

| 項目 | 仕様 |
|------|------|
| 埋め込みモデル | `gemini-embedding-001`（768次元） |
| 検索方式 | コサイン類似度 Top-K（デフォルト3件） |
| 最小類似度 | 0.0（設定可能） |
| フォールバック | ナレッジ0件 or 検索結果0件時、LLM 直接応答（デフォルトON） |

### ストレージ

| データ | ローカル | Firestore |
|--------|----------|-----------|
| ロール | SQLite `config` テーブル | `config/bot_role` ドキュメント |
| ナレッジ | ChromaDB `knowledge` コレクション | `knowledge` コレクション |

### 環境変数

| 変数 | 必須 | 説明 |
|------|------|------|
| GEMINI_API_KEY | ○ | Gemini API キー |
| LINE_CHANNEL_SECRET | ○ | LINE チャネルシークレット |
| LINE_CHANNEL_ACCESS_TOKEN | ○ | LINE チャネルアクセストークン |
| FIRESTORE_PROJECT_ID | △ | 設定時は Firestore 使用 |
| GEMINI_MODEL | - | デフォルト: gemini-1.5-flash |
| GEMINI_EMBED_MODEL | - | デフォルト: gemini-embedding-001 |
| RAG_TOP_K | - | 検索件数（デフォルト: 3） |
| RAG_MIN_SIMILARITY | - | 最小類似度（デフォルト: 0.0） |
| RAG_FALLBACK_TO_LLM | - | フォールバック有効（デフォルト: true） |

## 処理フロー概要

1. ユーザーが LINE でメッセージ送信
2. LINE Platform が Webhook を呼び出し
3. 署名検証 → イベント解析（message, text のみ対象）
4. 重複排除（reply_token TTL 600秒）
5. ロール取得 → ナレッジ件数判定
6. ナレッジあり: 埋め込み → 類似度検索 → プロンプト構築 → Gemini 生成
7. ナレッジなし or フォールバック: ロール + ユーザー入力を直接 Gemini に渡す
8. LINE API で返信
